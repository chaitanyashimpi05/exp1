Computer systems store user passwords in a special file called the shadow file (/etc/shadow).
To protect users, passwords are not saved in plain text. Instead, they are converted into a scrambled form called a hash using cryptographic hashing algorithms like:

MD5 ‚Üí $1$

SHA-256 ‚Üí $5$

SHA-512 ‚Üí $6$

yescrypt ‚Üí $y$ (used in newer Linux systems)

Hashing is one-way, meaning the original password cannot be directly recovered.

To check the strength of passwords and to perform security auditing, tools like John the Ripper (JtR) are used.
John the Ripper is an open-source password-cracking tool. It works by:

Reading password hashes from files like /etc/shadow.

Applying different attacks such as:

Dictionary Attack ‚Äî tries passwords from a wordlist (like rockyou.txt).

Brute Force Attack ‚Äî tries all possible combinations.

Rule-based Attack ‚Äî modifies dictionary words (adding numbers, symbols).

Comparing generated hashes to the actual hash in the shadow file.

If the hashes match, the password is ‚Äúcracked.‚Äù

How Linux stores passwords.

How weak passwords can be cracked easily.

Why strong hashing algorithms and strong passwords are necessary.

Why system administrators must use secure hashing methods and strong password policies to protect user accounts.
